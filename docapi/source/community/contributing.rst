Contributing
============

Thank you for your interest in contributing to **aws-iam-monitor**!

Contributions from the community are welcome and appreciated. Whether you're
reporting bugs, suggesting enhancements, improving documentation, or submitting
code, your help is valuable.

.. contents::
   :local:
   :depth: 2

Getting Started
---------------

1. Fork the repository.

2. Clone your fork:

  .. code-block:: bash

    git clone https://github.com/YOUR_USERNAME/aws-iam-monitor.git

3. Create a new branch for your changes:

  .. code-block:: bash

    git checkout -b your-branch-name

4. Make your changes and commit them.

5. Push your branch and open a Pull Request.

How to Contribute
-----------------

You can contribute in many ways, including:

* Fixing bugs.
* Improving documentation.
* Suggesting new features.
* Implementing enhancements.
* Writing or improving tests.
* Reviewing Pull Requests.

Please check the project's GitHub Issues page to see where help is needed.

Code Guidelines
---------------

When contributing code:

* Keep the code clean, readable, and maintainable.
* Follow the project's coding style.
* Write descriptive commit messages.
* Add tests for new functionality whenever appropriate.
* Ensure the project builds successfully.
* Verify that all tests pass before submitting a Pull Request.

Pull Request Process
--------------------

Before submitting a Pull Request:

#. Ensure your changes pass all existing tests.
#. Rebase or update your branch with the latest changes from the main branch.
#. Provide a clear and concise description of your changes.
#. Reference any related issues when applicable.
#. Respond to review feedback promptly.

Reporting Issues
----------------

When reporting a bug, please include:

* A clear and descriptive title.
* Steps to reproduce the issue.
* Expected behavior.
* Actual behavior.
* Relevant logs or screenshots.
* Operating system and software versions.

Code of Conduct
